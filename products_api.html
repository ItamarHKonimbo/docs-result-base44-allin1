Endpoints

GET /entities/Product

List Product records with optional filtering, sorting, and pagination.

ParameterLocationDescription
qqueryJSON query filter, e.g. {"status":"active"}
limitqueryMaximum number of records to return
skipqueryNumber of records to skip (pagination)
sort_byqueryField name to sort by. Prefix with - for descending, e.g. -created_date
 
const records = await base44.entities.Product.list();

POST /entities/Product

Create a single Product record.

const record = await base44.entities.Product.create({
  // your data
});

DELETE /entities/Product

Delete multiple Product records matching a query filter.

⚠️ WARNING: passing an empty {} filter deletes ALL records.
await base44.entities.Product.deleteMany({
  itemId: "Example itemId"
});

POST /entities/Product/bulk

Bulk create multiple Product records in a single request.

const records = await base44.entities.Product.bulkCreate([
  { /* record 1 */ },
  { /* record 2 */ },
]);

PUT /entities/Product/bulk

Bulk update Product records.

⚠️ Bulk-update is not available via SDK — use the REST API directly.

PATCH /entities/Product/update-many

Update many Product records matching a query.

⚠️ Update-many is not available via SDK — use the REST API directly.

GET /entities/Product/{Product_id}

Get a single Product record by its internal ID.

const record = await base44.entities.Product.get(recordId);

PUT /entities/Product/{Product_id}

Update a specific Product record by ID.

const record = await base44.entities.Product.update(recordId, {
  // fields to update
});

DELETE /entities/Product/{Product_id}

Delete a specific Product record by ID.

await base44.entities.Product.delete(recordId);

PUT /entities/Product/{Product_id}/restore

Restore a previously deleted Product record.

const record = await base44.entities.Product.restore(recordId);

הסבר מפורט: מה זה הקובץ הזה?

מה זה בכלל?

הקובץ הזה הוא תיעוד API של ה-entity בשם Product בתוך פלטפורמת Base44 — פלטפורמת no-code/low-code ישראלית שמאפשרת בניית אפליקציות עם בסיס נתונים מובנה ו-SDK מוכן לשימוש.

בפועל, ה-Product הוא טבלת מוצרים של חנות אונליין (ככל הנראה של Homebat / Konimbo), שמגיעה מספקים שונים. הקובץ מתאר:

  • אילו שדות יש לכל מוצר (Schema)
  • אילו פעולות API אפשר לבצע עליו (Endpoints)
  • איך לקרוא לאותן פעולות דרך ה-SDK של Base44

מה זה "Entity" ב-Base44?

Entity הוא בעצם מודל נתונים / טבלה בבסיס הנתונים של Base44. כל entity מקבל אוטומטית את כל ה-CRUD endpoints (Create, Read, Update, Delete) + endpoints מיוחדים כמו bulk ו-restore. ה-SDK של Base44 מאפשר לגשת אליהם בצורה נוחה מ-JavaScript/TypeScript.

בקצרה: base44.entities.Product זה כמו ORM — אובייקט שמאפשר לשאול ולשנות את טבלת המוצרים בלי לכתוב SQL.

מה יש בשדות ה-Schema?

השדות מחולקים לכמה קבוצות לוגיות:

1. זיהוי המוצר

  • itemId — ה-ID שמגיע מהספק (חובה). זה המפתח שמחבר בין המוצר בחנות לבין הספק.
  • id — ה-ID הפנימי של Base44 (נוצר אוטומטית).
  • friendlyurl — ה-slug של המוצר ב-URL (זהה ל-itemId).

2. מידע בסיסי על המוצר

  • name — שם המוצר בעברית (חובה).
  • category / subCategory — קטגוריה וקטגורית משנה בעברית.
  • itemModel / manufactModel — מספרי דגם (של הספק ושל היצרן).
  • manufactName / manufacturerURL — מידע על היצרן.
  • images — רשימת URLs של תמונות, מופרדות בנקודה-פסיק (;).
  • מפרט — טבלת HTML עם המפרט הטכני של המוצר.

3. מחיר ומלאי

  • costPrice — מחיר עלות מהספק.
  • regularPrice — מחיר קמעונאי מחושב (מחיר מכירה).
  • inventory — 100 = במלאי, -100 = אזל מהמלאי. (ערך בינארי, לא כמות אמיתית)
  • on_the_way_inventory — האם המוצר בדרך (הזמנה בתהליך).
  • last_price / last_price_date — המחיר האחרון הידוע + תאריכו (היסטוריה).

4. תנאי מכירה

  • warranty — אחריות בחודשים.
  • warrantyText — טקסט חופשי לאחריות.
  • paymentsNoCredit / maxPayments — תנאי תשלום.
  • deliveryTime — זמן אספקה.

5. ספקים

השדה supplierName הוא enum עם הספקים הקיימים במערכת: מור לוי, אול אין וואן, קומפיוטר סי דאטה, טונוס, מתקינט, מפעיל, טכנורצף. זה אומר שמוצרים שונים מגיעים ממקורות שונים — ויש כנראה תהליכי סנכרון אוטומטיים מכל ספק.

6. עיבוד שם המוצר (AI Pipeline)

תובנה חשובה: יש כאן pipeline של עיבוד שמות:
  • name — השם הגולמי שמגיע מהספק.
  • name_parsed — האם כבר עברנו parsing על השם.
  • שם מוצר מנותח — השם לאחר parsing / ניתוח.
  • name_committed — האם אישרנו את השם המנותח ופרסמנו אותו.
זה רומז לתהליך אוטומטי (כנראה AI) שמנתח שמות מוצרים גולמיים ומייצר שמות נקיים יותר.

7. שדות GMF (Google Manufacturer Feed)

כל השדות שמתחילים ב-GMF_ הם מפרטים טכניים מובנים שמיועדים ל- Google Manufacturer Feed — ממשק שמאפשר לשלוח מידע עשיר על מוצרים לגוגל לצורך Google Shopping. הם מכסים:

  • מפרט חומרה: מעבד, GPU, RAM, אחסון, Chipset
  • תצוגה: גודל מסך, סוג פאנל, שטוח/קעור
  • קישוריות: WiFi, LTE, סוג חיבור, כמות פורטים
  • כללי: גודל, אחריות, OS, סוג מוצר, תקציב

אילו פעולות אפשר לבצע? (Endpoints)

קריאת נתונים

GET /entities/Product — שאיבת רשימת מוצרים עם:
  • פילטור (q): JSON שמסנן רשומות, למשל {"supplierName":"טונוס"}
  • pagination (limit + skip): לדפדוף בין עמודים
  • מיון (sort_by): למשל -regularPrice = מחיר יורד
GET /entities/Product/{id} — שאיבת מוצר בודד לפי ה-ID הפנימי של Base44.

יצירת נתונים

POST /entities/Product — יצירת מוצר אחד.
POST /entities/Product/bulk — יצירת מספר מוצרים בבת אחת (יעיל לסנכרון ספקים).

עדכון נתונים

PUT /entities/Product/{id} — עדכון מוצר בודד לפי ID.
PUT /entities/Product/bulk — עדכון מרובה (דרך REST בלבד, לא SDK).
PATCH /entities/Product/update-many — עדכון כל הרשומות שעומדות בתנאי שאילתה (דרך REST בלבד).

מחיקה ושחזור

DELETE /entities/Product — מחיקה לפי שאילתה. ⚠️ זהירות: {} ריק = מוחק הכל!
DELETE /entities/Product/{id} — מחיקת מוצר בודד.
PUT /entities/Product/{id}/restore — שחזור מוצר שנמחק (Base44 שומר מחוקים).

מה התוצאה שמתקבלת מהקריאה?

קריאת list() מחזירה מערך של אובייקטים, כל אחד מהם רשומת מוצר עם כל השדות שהוגדרו ב-Schema. קריאת get(id) מחזירה אובייקט בודד. קריאות create / update מחזירות את הרשומה המעודכנת.

דוגמה לאובייקט שמוחזר:

  "id": "abc123",            // Base44 internal ID
  "itemId": "BT-9900X",     // Supplier ID
  "name": "מחשב נייד ASUS VivoBook 15",
  "regularPrice": 3299,
  "costPrice": 2800,
  "inventory": 100,          // 100 = in stock
  "supplierName": "טונוס",
  "category": "מחשבים ניידים",
  "warranty": 24,
  "images": "https://img1.jpg;https://img2.jpg",
  "GMF_מעבד": "Intel Core i5-12th Gen",
  "GMF_נפח זיכרון ראם": "16GB",

מה הזרימה הכללית?

מה שאפשר להבין מהמבנה הזה:

  1. ספקים שולחים נתוני מוצר → נשמרים ב-Base44 עם itemId של הספק.
  2. מחירים ומלאי מתעדכנים דרך updated_at ו-last_price.
  3. Pipeline של שמות: שם גולמי → parsed → committed.
  4. מפרט GMF מאפשר לשלוח את הנתונים לגוגל שופינג.
  5. הקובץ הזה מאפשר לקוד בצד ה-client (Next.js / React) לשאוב ולעדכן מוצרים.

מה אמורים להבין מהקובץ?

✅ השדות החשובים ביותר לעבוד איתם:
  • itemId — המזהה הנדרש ליצירה, ה-link לספק
  • inventory — 100 / -100 בלבד, לא כמות
  • regularPrice — המחיר שמוצג ללקוח
  • סטטוס מוצר — boolean אם המוצר מוצג בחנות
  • images — URLs מופרדים ב-;
⚠️ נקודות שצריך לשים לב אליהן:
  • שני endpoints לא זמינים דרך SDK: bulk PUT ו-update-many PATCH — חייבים REST API.
  • מחיקה עם {} ריק = מוחק את כל המוצרים.
  • מפרט מכיל HTML גולמי — צריך dangerouslySetInnerHTML בריאקט.
  • שדות בעברית (סטטוס מוצר, מפרט, שם מוצר מנותח) — צריך quotation בג'אווסקריפט.
🔴 הגבלות ידועות:
  • אין תיעוד על rate limits.
  • אין תיעוד על authentication — כנראה מנוהל ברמת ה-Base44 app.
  • inventory הוא ערך בינארי בלבד — אין מידע על כמות מדויקת במלאי.
